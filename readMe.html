Writing a good readMe file.

A readMe file is file which contains information about other files in directory or archive of computer software.
Or, in other words it is simply a form of documentation that introduces and explains a project.It contains information that is commonly required to understand what the project is about.

It is usually a simple plain text file called README. It can be in the format of .TXT or .md where .txt stands for text format and md for markdown. However a .md file is used mostly for making a README file.

Why should I make it?

You should make it because it's an easy way to answer questions that your audience will likely have regarding how to install and use your project and also how to collaborate with you.

Where should I put it?

Putting a README file can be bit tricky that is you should always put this file in the top level directory of the project because this is where someone who is new to your project will start out. Code hosting services also looks for your README file and display it along with the list of files and directories in your project.
